■ プロジェクト名

運用ログ解析・レポート自動作成ツール

■ プロジェクト概要

本ツールは、サーバーの運用ログをもとに
障害傾向やリソース使用状況を自動で分析し、
レポートとして可視化することを目的としたツールである。

ログの集計・分析・レポート作成を自動化することで、
運用業務の効率化と品質向上を実現する。

■ 開発目的

日常運用で発生するログ確認・集計作業の自動化

障害や異常傾向の早期発見

運用レポート作成工数の削減

Pythonによるデータ処理・自動化スキルの向上

■ 機能概要
① ログ読込機能

CSV形式およびJSON形式のログファイルを読み込み可能

複数ファイルの同時処理に対応

指定フォルダからの一括読込機能

② データ整形・クレンジング機能

重複データの除去

必須項目が不足しているデータの除外

日付形式の統一処理

不正データのフィルタリング

③ 集計・分析機能

サーバー別・日別に以下の情報を集計する。

ログレベル別件数（INFO / WARNING / ERROR）

CPU使用率の平均値

メモリ使用率の平均値

異常発生傾向の分析

④ レポート出力機能

Excel形式で集計結果を出力

サーバーごとにシートを分割

日別集計データを一覧化

自動レポート生成機能

⑤ グラフ可視化機能

CPU・メモリ使用率の推移を折れ線グラフで表示

エラー件数を棒グラフで表示

画像ファイル（PNG形式）として保存

⑥ （拡張機能・予定）

FlaskによるWeb画面表示

ファイルアップロード機能

リアルタイム分析対応

メール通知機能

■ 使用技術

言語：Python

データ処理：Pandas

Excel操作：OpenPyXL

可視化：Matplotlib / Seaborn

Web（予定）：Flask

バージョン管理：Git / GitHub

■ 想定利用シーン

日次運用レポートの自動作成

障害分析資料の作成

運用改善のためのデータ分析

チーム内共有用レポート作成

■ 本プロジェクトのアピールポイント（面接用）

実務を想定した課題設定

運用業務の自動化を意識した設計

データ処理から可視化まで一貫して実装

拡張性を考慮した構成
